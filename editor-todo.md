# 编辑器待补功能清单

## 🟡 中优先级

### 1. 查找替换

**必要性：** 长文档改术语、批量修正必备。目前只能手动逐处修改。

**实现思路：**
- `@tiptap/extension-search-and-replace`（社区维护）或自己基于 ProseMirror `search` 插件做
- 工具栏按钮 → 展开查找替换栏（浮在编辑器顶部）
- 支持：查找、替换、全部替换、区分大小写
- Operate 类有现成参考

**复杂度：** 中（如果有现成插件则低）

---

### 2. 文字对齐

**必要性：** 手册中表格内容居中、标题居中很常见。目前无法设置对齐。

**实现思路：**
- 安装 `@tiptap/extension-text-align`，零配置
- 工具栏：左对齐 / 居中 / 右对齐 / 两端对齐 四个按钮
- 图标：`i-lucide-align-left`、`i-lucide-align-center`、`i-lucide-align-right`、`i-lucide-align-justify`

**复杂度：** 低（10 分钟）

---

### 3. 水平分隔线

**必要性：** 长文档划分章节、分隔不同主题。

**实现思路：**
- `@tiptap/extension-horizontal-rule`（StarterKit 可能已包含）
- 工具栏一个按钮，插入 `<hr>`

**复杂度：** 低（5 分钟）

---

### 4. 下划线

**必要性：** 基本文本格式，配合加粗/斜体/删除线形成完整格式组。

**实现思路：**
- `@tiptap/extension-underline`
- 工具栏 `U` 按钮（放在 `S` 删除线旁边）

**复杂度：** 低（5 分钟）

---

## 🟢 低优先级

### 5. 待办清单

**必要性：** 操作手册中"检查清单""操作步骤"场景实用。

**实现思路：**
- `@tiptap/extension-task-list` + `@tiptap/extension-task-item`
- 工具栏 checkbox 按钮

**复杂度：** 低

---

### 6. 编辑器底部状态栏

**必要性：** 用户不知道自己写了多少内容，当前在哪个标题层级。

**实现思路：**
- 纯前端，在编辑器底部加一行状态栏
- 显示：字数 / 字符数 / 当前段落格式（如"标题2"）
- 用 `editor.storage.characterCount` 或自己统计

**复杂度：** 低

---

### 7. 文字颜色 / 高亮

**必要性：** 标注重点内容。但容易滥用导致手册风格不统一。

**实现思路：**
- `@tiptap/extension-highlight`（背景高亮）
- `@tiptap/extension-color`（文字颜色）
- 工具栏颜色选择器（可用现有 ColorPicker 或简单预设色板）

**复杂度：** 中（主要是颜色选择器 UI）

---

### 8. 全屏写作模式

**必要性：** 专注编写，隐藏侧边栏和顶部导航。

**实现思路：**
- 纯前端：编辑器容器加一个 `fullscreen` 状态
- 全屏时：编辑器 `fixed inset-0 z-50`，左侧导航树可折叠
- Esc 退出全屏
- 工具栏添加全屏按钮

**复杂度：** 低（纯 CSS + 状态切换）

---

## ⚪ 可选

### 9. Markdown 快捷输入

StarterKit 可能已部分支持（如 `# 空格` → 标题）。可以检查并补全：
- `> 空格` → 引用
- `- 空格` → 无序列表
- `1. 空格` → 有序列表
- `` ` 代码 ` `` → 行内代码
- `---` → 分隔线

**复杂度：** 低（如果 StarterKit 已支持则无需改动）

---

### 10. 图片尺寸调整

目前插入图片后无法调整显示大小。

**实现思路：**
- TipTap Image 扩展配置 `allowBase64: true` 后可拖拽调整
- 或选中图片后弹出尺寸输入框

**复杂度：** 中

---

### 11. 打印样式优化

**必要性：** 浏览器直接打印（Ctrl+P）手册时排版可能不好。

**实现思路：**
- 添加 `@media print` 样式
- 隐藏工具栏、导航栏
- 调整页边距和字体大小

**复杂度：** 低

---

## 优先级建议

如果下次集中补编辑器功能，推荐顺序：

```
1. 下划线         ← 5 分钟，补全基础格式
2. 水平分隔线      ← 5 分钟，即装即用
3. 文字对齐        ← 10 分钟，手册刚需
4. 查找替换        ← 核心功能，长文档必备
5. 底部状态栏      ← 提升编辑体验
```

前三项加起来不到半小时，第四项值得单独花时间做好。
