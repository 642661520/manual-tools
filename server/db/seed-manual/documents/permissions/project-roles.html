项目角色详解

项目角色控制用户在特定项目内的操作权限。系统 Admin 不会自动继承项目角色，必须被显式添加为项目成员后才可操作该项目内容。

PM（项目负责人）

  • 管理项目设置和审核链
  • 管理项目成员
  • 创建和管理分类、功能、目录
  • 指派章节编辑者
  • 审核文档（通过/退回/重置状态）
  • 预览和发布手册版本
  • 管理版本生命周期和可见性
  • 项目数据导入导出
  • 编辑文档内容

Writer（编辑者）

  • 查看项目中的所有功能和分类
  • 编辑文档内容（富文本编辑器）
  • 参与协同编辑
  • 提交文档审核
  • 查看手册预览

Viewer（查阅者）

  • 查看功能列表和分类
  • 查看文档内容（只读模式）
  • 查看手册预览和已发布版本
  • 不能被指派为编辑者

权限隔离原则

  • 项目管理操作（成员管理、审核链配置、导入导出）：Admin 保留全局管理权
  • 内容流水线操作（编辑、审核、状态流转、指派编辑者、发布）：仅项目 PM/Writer 可操作，Admin 也需显式加入项目
  • 编辑者指派下拉菜单仅显示 Writer 角色成员，Viewer 和 PM 不可见（PM 无需被指派即可操作所有内容）
