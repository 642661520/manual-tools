项目导出

支持将整个项目的数据导出为 ZIP 文件，用于数据迁移和备份。

导出内容

导出 ZIP 包含：

  • 项目基本信息（名称、描述、审核链配置）
  • 所有分类和功能定义
  • 所有文档内容（Y.js 增量数据和快照）
  • 所有目录和发布版本
  • 文档中引用的上传文件（图片、视频）

操作步骤

  1. 进入项目设置 → 数据管理
  2. 点击导出项目
  3. 系统估算导出大小并显示预览
  4. 确认后开始导出（异步任务）
  5. 导出完成后下载 ZIP 文件

任务管理

导出是异步任务，支持进度查询和历史记录查看。导出任务 24 小时后自动过期清理。

图片占位：data-import-export - export

项目导入

支持从 ZIP 文件导入项目数据，可用于数据迁移和恢复。

导入流程

  1. 进入项目设置 → 数据管理
  2. 点击导入项目，选择 ZIP 文件上传
  3. 系统分析 ZIP 内容并与当前项目对比
  4. 在差异确认界面查看将新增和冲突的条目
  5. 对冲突条目选择处理策略（覆盖/跳过）
  6. 确认后执行导入

冲突处理

当导入数据中的实体（分类、功能、目录、文档）ID 与当前项目冲突时，您可以为每个实体单独选择覆盖或跳过策略。

图片占位：data-import-export - import

差异分析

导入前系统会自动进行差异分析，让您清楚了解导入会带来哪些变化。

分析结果

差异报告显示：

  • 新增 — 导入数据中有但当前项目中没有的条目数量
  • 冲突 — 两方都有的条目（按 ID 匹配）
  • 上传文件 — 将复制到上传目录的文件数量和总大小

决策辅助

对于每个冲突条目，系统会显示源名称和目标名称，帮助您做出跳过或覆盖的决策。
