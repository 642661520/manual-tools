章节管理

每个功能由一个或多个章节（Section）组成。章节是文档编写的最小单位。

章节属性

  • Key（标识符） — 系统自动生成 8 位随机十六进制字符，用于构成文档 ID（如 feature-id/a1b2c3d4）。Key 创建后不可修改
  • Title（标题） — 在功能详情和目录导航中展示。标题创建后不可直接修改，需要删除并重新添加

操作说明

在功能的编辑对话框中，可以：

  • 添加章节 — 点击输入框，输入标题后按回车或点击添加按钮，系统自动生成 key
  • 删除章节 — 点击章节右侧的删除按钮，注意该章节的文档会变为孤立状态
  • 排序章节 — 拖拽章节左侧的拖拽手柄调整在功能中的展示顺序

孤立文档处理

当章节被删除后，其已有文档不会自动删除，而是标记为"孤立文档"。在功能列表中会显示带有警告图标的橙色标记，PM 可以查看并手动删除这些孤立文档。

默认章节

如果创建功能时未添加任何章节，系统会在首次编辑时自动创建一个 _default 默认章节，方便快速开始编写。
