自动构建

每次发布版本后，系统会自动生成静态文档站点。

构建内容

发布的静态站点包含：

  • 完整手册内容的 HTML 页面
  • 左侧导航边栏和章节目录
  • 版本切换器（可切换到其他已发布版本）
  • 全文搜索功能（前端倒排索引）
  • 响应式布局样式

存储位置

静态站点文件存储在 data/docs/{catalogId}/{version}/ 目录下。

触发方式

发布版本时自动触发构建。如果需要手动重建（如修改了站点模板），可以在版本管理中重新发布。

站点访问

已发布的文档站点通过 /docs/ 路径访问。

访问地址

格式为 /docs/{catalogId}/。访问 /latest/ 会自动重定向到最新版本。站点支持：

  • 版本切换 — 在已发布的版本之间切换
  • 搜索 — 在当前版本中全文搜索文档内容
  • 导航 — 左侧边栏展示手册目录结构

访问控制

根据发布时设置的可见性级别，未登录用户或非项目成员访问受限制的文档时会看到登录提示页面。

更新机制

静态站点会随着版本发布自动更新。

更新流程

  1. 文档内容修改和审核
  2. 发布新版本
  3. 系统自动重新构建静态站点
  4. 新版本在 /latest/ 路径下立即可用
  5. 旧版本站点保持不变，通过版本切换器仍可访问

版本共存

多个版本的静态站点可以同时存在并独立访问。用户可以随时切换到以前的版本查看历史文档。

离线站点导出

当手册需要部署到内网环境时，可以将所有公开版本的静态站点打包下载，方便在 nginx 等静态服务器上一键部署。

导出方式

在手册预览页面的工具栏中，点击 下载离线站点 按钮即可下载 ZIP 包。该操作属于 catalog 级别，会导出当前目录下所有已发布的公开版本，不依赖当前选择的版本。

ZIP 包内容

下载的 ZIP 包解压后目录结构如下：

docs/{catalogId}/
├── index.html          # 自动跳转到最新公开版本
├── versions.json       # 公开版本列表
└── v1.0/
    ├── index.html      # 封面页
    ├── ch01.html       # 各章节页面
    ├── ch02.html
    └── search-index.json

部署方式

解压到 nginx 的 web root 目录即可使用，无需额外配置：

server {
    listen 80;
    root /var/www/manual-site;
    index index.html;
}

部署后访问 /docs/{catalogId}/ 自动跳转到最新版本，版本切换器和全文搜索均可正常使用。

适用场景

  • 将公开手册分发到内网环境供离线查阅
  • 对方无需部署本平台，仅需一个静态文件服务器
  • 导出的 ZIP 包含所有公开版本，支持版本回看
