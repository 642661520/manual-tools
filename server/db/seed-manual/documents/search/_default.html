搜索使用

平台提供基于 SQL LIKE 的全文搜索功能，支持跨文档内容搜索，索引在服务器启动时自动重建。

搜索入口

使用快捷键 Ctrl+K 或点击顶部导航栏的搜索按钮打开搜索对话框，输入关键词即可搜索当前项目中所有文档的内容。

搜索范围

搜索覆盖所有文档章节的标题和正文内容。搜索结果按匹配顺序显示，包含匹配的章节标题和内容片段，点击结果可直接跳转到对应章节的编辑页面。

搜索权限

仅项目成员可以搜索该项目内的文档内容，确保数据安全。

静态站点搜索

已发布的静态文档站点也内建了搜索功能，使用前端倒排索引实现离线搜索，无需后端服务。

图片占位：搜索对话框：使用 Ctrl+K 打开，输入关键词显示搜索结果

索引重建

搜索索引在服务器启动时自动重建，确保搜索结果的准确性。

自动重建

每次服务器启动时，系统会：

  1. 清空现有搜索索引
  2. 读取所有项目的所有文档内容
  3. 重建搜索索引

手动重建

管理员可以通过 API POST /api/v1/search/rebuild（请求体传入 { projectId: "xxx" }）手动触发指定项目的搜索索引重建，无需重启服务器。

索引性能

索引重建的时间取决于文档总量，通常在几秒到几十秒之间完成。
