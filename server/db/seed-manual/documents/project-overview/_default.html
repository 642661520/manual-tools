项目概念

在 Manual Tools 平台中，项目（Project）是数据组织和隔离的基本单位。每个项目拥有独立的：

  • 分类系统 — 自定义的功能分类和颜色标记
  • 功能集合 — 文档编写的功能骨架和章节结构
  • 文档内容 — 基于 Y.js 协同编辑的富文本文档
  • 目录编排 — 将功能组装为完整手册的目录
  • 成员权限 — 独立的项目成员和角色管理

平台首次启动时会自动创建一个默认项目，包含本使用手册的完整内容，方便您快速上手。

项目的作用

通过多项目机制，您可以：

  1. 为不同的产品线或业务模块创建独立项目
  2. 在同一平台中管理多个文档工程
  3. 按项目粒度控制用户的访问和操作权限
  4. 独立导出和迁移单个项目的数据

多项目隔离

Manual Tools 采用严格的项目级数据隔离机制：

数据隔离范围

  • 分类 — 每个项目的分类完全独立，不可跨项目共享
  • 功能 — 功能隶属于特定项目，不能跨项目引用
  • 文档 — 文档内容通过功能间接关联到项目
  • 目录 — 目录和发布版本按项目隔离
  • 成员 — 用户需要被添加到项目中才能访问项目数据

切换项目

页面顶部导航栏提供项目选择器，点击即可在不同项目之间切换。切换项目后，功能列表、目录编排等页面会自动加载对应项目的数据。当前选择的项目会持久化到浏览器本地存储，下次访问时自动恢复。

安全性

所有 API 请求都会校验用户对目标项目的访问权限。即使用户知道其他项目的 ID，没有项目成员身份也无法查看或操作其中的数据。
