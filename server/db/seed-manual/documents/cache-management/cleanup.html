缓存清理

系统提供手动和自动两种缓存清理方式。

自动清理

服务器启动时自动执行：

  • 扫描 data/cache/exports/ 目录，删除超过 EXPORT_CACHE_TTL_DAYS 天的导出缓存
  • 扫描 data/cache/remote/ 目录，删除超过 REMOTE_CACHE_TTL_DAYS 天的远程资源缓存

手动清理

在数据管理页面可以手动执行清理操作：

  1. 进入系统设置 → 数据管理
  2. 点击清理过期缓存按钮，立即清理所有过期缓存文件
  3. 或使用各缓存区域的单个删除功能精确清理

清理时机建议

  • 日常运行 — 依赖自动清理即可，无需手动干预
  • 磁盘空间不足 — 手动清理过期缓存释放空间
  • 内容更新后 — 使特定目录的导出缓存失效，确保下次导出使用最新内容

注意事项

  • 清理缓存不会影响已发布的文档站点
  • 已转换为本地路径的文档图片不会因远程缓存清理而丢失
  • 清理后首次导出或访问远程资源时需要重新生成/下载
