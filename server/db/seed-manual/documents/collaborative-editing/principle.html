协同原理

Manual Tools 使用 Y.js 作为协同编辑引擎，实现多人实时协作。

Y.js 简介

Y.js 是一个基于 CRDT（Conflict-free Replicated Data Type，无冲突复制数据类型）算法的高性能协同框架。它的核心优势是：

  • 无冲突 — 多个用户同时编辑同一文档时，CRDT 算法自动合并所有修改，不会产生冲突
  • 离线支持 — 用户可以在离线状态下编辑，网络恢复后自动同步
  • 去中心化 — 不需要中心服务器做冲突仲裁
  • 增量同步 — 只传输编辑的增量数据，而非整个文档

技术架构

协同编辑的数据流：

  1. 用户在 TipTap 编辑器中编辑内容
  2. 编辑操作通过 Y.js 绑定转换为 CRDT 更新
  3. 更新通过 WebSocket 连接发送到服务器
  4. 服务器持久化更新到 SQLite 数据库
  5. 服务器广播更新给同一文档的其他在线用户
  6. 其他用户的编辑器接收更新并自动应用

数据存储

每个文档的数据以两种形式存储：

  • 增量更新（Updates） — 每次编辑的增量数据
  • 完整快照（Snapshots） — 每 500 次更新后自动生成一次完整快照，加速文档加载
